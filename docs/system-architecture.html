sequenceDiagram
    participant C as Customer

    participant STR as Stripe
    participant WH as /api/webhooks/stripe

    participant SS as ShipStation
    participant EMAIL as Resend Email

    C->>SHOP: Browse Products
    C->>CART: Add to Cart
    CART->>API: POST /api/checkout {items, email, couponCode}
    API->>DB: Validate inventory
    API->>DB: Validate coupon
    API->>STR: createCheckoutSession()
    STR-->>API: session.url
    API-->>CART: Redirect to Stripe

    C->>STR: Complete Payment
    STR->>WH: checkout.session.completed
    WH->>DB: Create Order + OrderItems
    WH->>DB: Decrement Inventory
    WH->>SS: processOrderForShipping()
    WH->>EMAIL: sendOrderConfirmation()
    WH->>DB: Update status to PAYMENT_CONFIRMED

    SS->>WH: SHIP_NOTIFY webhook
    WH->>DB: Update tracking + SHIPPED status
    WH->>EMAIL: sendShippingNotification()

7. Security Architecture

Middleware Security Headers

  • X-Frame-Options: DENY - Anti-clickjacking
  • X-Content-Type-Options: nosniff
  • X-XSS-Protection: 1; mode=block
  • Referrer-Policy: strict-origin-when-cross-origin
  • Permissions-Policy: camera=(), mic=(self)
  • HSTS: max-age=63072000; preload

Content Security Policy

  • default-src 'self'
  • script-src 'self' 'unsafe-inline' 'unsafe-eval'
  • img-src 'self' data: blob: *.wpengine.com
  • connect-src 'self' ssapi.shipstation.com
  • frame-ancestors 'none'

Authentication

  • PBKDF2 password hashing
  • JWT-style session tokens (HMAC-SHA256)
  • httpOnly + Secure + SameSite cookies
  • Token-based password reset (1hr expiry)
  • Rate limiting on all endpoints

Bot Protection

  • Cloudflare Turnstile on register, contact, newsletter
  • Rate limiting: 60 req/min API, 10 req/min webhooks
  • Webhook HMAC signature verification (Stripe + ShipStation)
  • Admin routes: role-based access control
